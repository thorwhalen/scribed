scribed.tools

Command-line tools for scribed (dispatched via argh in __main__).

Each function here is a thin, CLI-friendly wrapper over the Python API; argh turns their signatures into subcommands and options. Run scribed <command> --help (after pip install 'scribed[cli]') or python -m scribed <command>.

scribed.tools.backends(*, capability: str | None = None)[source]

List the backends scribed can run right now (optionally by capability).

Parameters:

capability – Filter to a capability, e.g. diarize, stream, translate.

scribed.tools.doctor()[source]

Report which backends are usable now, and how to install the rest.

scribed.tools.find(*, local: bool = False, remote: bool = False, free: bool = False, implemented: bool = False, diarization: bool = False, streaming: bool = False, word_timestamps: bool = False, language: str | None = None)[source]

Filter the ledger and print matching backends (id, where, pricing, best-for).

Flags compose (AND). Example: scribed find --local --free --diarization.

Parameters:

  • local – Keep only backends that run locally.

  • remote – Keep only hosted/remote backends.

  • free – Keep only free / open-source backends.

  • implemented – Keep only backends scribed can run today.

  • diarization – Keep only backends that label speakers.

  • streaming – Keep only backends that support real-time streaming.

  • word_timestamps – Keep only backends that emit word-level timestamps.

  • language – Keep only backends whose languages mention this name/code.

scribed.tools.info(backend_id: str)[source]

Print a backend’s full ledger record as JSON.

Parameters:

backend_id – A ledger id, e.g. deepgram (see scribed find).

scribed.tools.install(backend_id: str, *, gpu: bool = False, yes: bool = False)[source]

Plan (default) or run (--yes) the pip install for a backend.

Without --yes it prints the plan and changes nothing. System deps and GPU wheels are surfaced, not run automatically.

Parameters:

  • backend_id – Backend id to install, e.g. faster-whisper.

  • gpu – Surface GPU-wheel guidance.

  • yes – Actually run pip install (otherwise just print the plan).

scribed.tools.requirements(backend_id: str, *, gpu: bool = False)[source]

Show what a backend needs to run (pip, system deps, GPU, weights, creds).

Parameters:

  • backend_id – Backend id, e.g. whisper.

  • gpu – Include GPU-wheel guidance.

scribed.tools.scaffold(backend_id: str, *, dest: str | None = None)[source]

Generate a new backend package from its ledger entry.

Parameters:

  • backend_id – The ledger id to scaffold (e.g. speechmatics).

  • dest – Optional destination directory.

scribed.tools.status(*, level: str = 'all', run_tests: bool = False, names: bool = False)[source]

Print an ASR-backend readiness table (levels: all ⊇ implemented ⊇ set_up ⊇ tested).

Parameters:

  • level – Restrict rows to a level: all | implemented | set_up | tested.

  • run_tests – Actually transcription-test the set-up backends — makes real API calls for set-up remotes (off by default, so a plain scribed status never bills you).

  • names – Also print, per level, a comma-separated Name (website) list.

scribed.tools.transcribe(audio: str, *, backend: str | None = None, language: str | None = None, diarize: bool = False, output: str = 'text')[source]

Transcribe an audio file/URL and print the result.

Parameters:

  • audio – Path or http(s) URL to the audio.

  • backend – Backend id (default: first installed). See scribed backends.

  • language – Language code hint, e.g. en (omit to auto-detect).

  • diarize – Ask the backend to label speakers (if it supports it).

  • output – text (default), json (text + segments), srt, or vtt.

scribed.tools.validate(backend_id: str)[source]

Smoke-test a backend adapter end to end and print the report.

Parameters:

backend_id – The backend id to validate (e.g. faster-whisper).
